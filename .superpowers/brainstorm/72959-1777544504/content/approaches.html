Tracking Fix — 3 Approaches

Root cause: ByteTrack loses players after ~1.5s occlusion → new UUID assigned → 60 phantom "players". Pick your preferred fix strategy.

A

Parameter Tuning Only

Increase lost_track_buffer 30 → 120 frames (6 sec buffer), lower IoU threshold 0.8 → 0.65, hard-cap tracks at 16. Simple config change, no new code.

Pros

  • Minimal risk
  • Fast to ship
  • No pipeline changes

Cons

  • Still fails on long occlusions (timeouts, rotations off-camera)
  • Doesn't fix existing data — re-analysis still produces some duplicates
B

Tuning + Post-processing Track Merger ★ Recommended

Tune parameters (like A), then after the full pipeline run, add a track merge pass: pair up tracks that are on the same team, never overlap in time, and whose last/first positions are spatially close → merge into one Player UUID. Then prune: keep only top-16 players by total frames seen.

Pros

  • Handles long occlusions + camera pans
  • Hard cap guarantees ≤16 players in output
  • Stats consolidate onto merged player
  • Re-analysis produces clean data

Cons

  • ~1–2 days extra work vs A
  • Merge heuristic has edge cases (e.g. two players in same position)
C

ByteTrack + Appearance ReID

Add a lightweight ReID model (e.g. OSNet or YOLO pose features) that extracts a jersey-color + body-shape embedding per player. Returning players matched by embedding similarity, not just IoU — survives arbitrary disappearances.

Pros

  • Most robust — handles any occlusion length
  • Industry-standard approach (used in BoT-SORT, StrongSORT)

Cons

  • New model to integrate (~1 week)
  • Slower pipeline (extra inference per frame)
  • Volleyball jerseys often similar → embedding confusion possible

Regardless of approach chosen, also includes:

  • 🔁 Re-analyze button in match detail UI — clears old tracking data, re-runs pipeline, shows WebSocket progress
  • 🎯 Team assignment fix — cluster players by court side (left=Team A, right=Team B) instead of guessing
  • 🔢 Stable display IDs — show sequential #1–#12 per team instead of raw UUIDs
